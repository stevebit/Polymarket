## Operational notes / gotchas

- **Azure Postgres firewall.** Drops happen when VPN switches your public
  IP; the symptom is `PoolTimeout` or `consuming input failed: server
  closed the connection unexpectedly`. Fix by whitelisting the current
  IP, then rerun. Code already retries transient drops per day.
- **PowerShell parsing.** Don't paste prose or log lines into the prompt
  — `<`, `>`, `:`, em-dashes, leading times like `02:11:55` all get
  parsed as commands or redirection. Use `#` for inline notes.
- **`.env`** holds `WEATHER_POSTGRES_URL` (Azure) and other secrets.
  Never read it back in chat or commit it.
- **Concurrent reads** against the same Postgres are fine (we ran multiple
  ingests in parallel with `predict_history`). Avoid running
  `fit_postprocess` while a replay is using `postprocess_coefs`.
- **Resume helpers.** To find the next `predict_history --start` after a
  partial run, query `predictions` for the last `target_date` with
  `n_stations = 11` at the chosen UTC cutoff hour; resume on the next
  calendar day. We deleted the helper script but the SQL pattern is in
  the chat history if you need it back.
